# DEER Post-Processing Tool
This repository is a Python package for expediting unit energy savings calculations from DEER prototypes.

Created by: Solaris Technical LLC

## Description
Using the simulation results generated by [DEER-Prototype-EnergyPlus](https://github.com/sound-data/DEER-Prototypes-EnergyPlus), this tool simplifies the post-processing method used to generate savings per the [Script](https://github.com/sound-data/DEER-Prototypes-EnergyPlus/tree/main/scripts) folder in the DEER-Prototypes-EnergyPlus repository. This method includes similar SQL queries and the lookup tables (i.e. building weights, norm units) to calculate final energy savings.

This tool is a package which contains a simple graphic user interface, Python script, and SQL queries which calculate the final energy savings based on the energy usage file (simdata.csv) produced by result2.py, a Solaris script currently used in place of result.py found in the Scripts folder above.

## Instructions
To run the tool, download the GitHub repository and collect the necessary inputs as stated below.

Additionally, Python or Anaconda must be installed to run the tool script.

### Tool Inputs
This image of the user interface shows the required information and files necessary to use the post-processing tool.
![](https://github.com/simularis/deer-ues-tool/blob/main/UI/gooey_1.jpg)
#### Manual Inputs
The tool requires the following manual inputs:
1. Measure Name: The name/number of the measure package in the format **SWXX0XX**.
2. Measure Type: The type of measure being processed based on the measure end-use. Dropdown options are **Cooling Capacity**, **PTAC/PTHP**, **Ceiling Insulation**, **Wall Insulation**, **Refrigerator/Freezer**, and **Whole House Fan**.
3. Sector: The measure pacakge sector. Dropdown options are **Residential** and **Commercial**.
4. Normalizing Unit: The normalizing unit which will be used to normalize the measure savings. Dropdown options are **Cap-Tons**, **Area-ft2-BA**, **Area-ft2**, **Household**, **kWhreduced**.

#### File Inputs
The tool requires the following input files:
1. Simdata: Csv file generated by result2.py
3. Meassure Definition: Csv file created per eTRM Offering ID table.
